No.1 of Top 10 Java Design Pattern

Java OOD
Publish Date:   2020-05-24

第一篇文章分享Java里 个人认为最常用、最重要的十个设计模式中的第一个:单例


单例Singleton
Level：Hard
Hard 原因很简单，看似最简单最基础的设计模式却拥有最多的变种

Singleton Pattern
restricts the instantiation of a class and ensures that only one instance of the class exists in the Java virtual machine.

My thought:

    1. make only one instance in jvm -> static constructor
    1. also need to restrict instantiation from other
  • -> private constructor
    1. provide a global access point to get the instance
  • -> public getMethod (private variable based on encapsulation principle)
    1. final? The final modifier allows the Java compiler and runtime
  • to make good optimization and thread-safety decisions.

1. Eager initialization


class EagerInit {
    private static final EagerInit instance = new EagerInit();

    private EagerInit() {
    }

    public static EagerInit getInstance() {
        return instance;
    }
}

2. Static block initialization

avoid create instance more than one time and good for error handle


class StaticBlockSingleton {
    private static StaticBlockSingleton instance;

    private StaticBlockSingleton() {
    }

    static {
        try {
            instance = new StaticBlockSingleton();
        } catch (Exception e) {
            throw new RuntimeException("Exception occurred in creating singleton instance");
        }
    }

    public static StaticBlockSingleton getInstance() {
        return instance;
    }
}

Eager and static block both create the instance of singleton
even if the class is not being used so far (not lazy initialization)

3. Lazy Initialization

create instance when we call get method


class LazyInitializedSingleton {

    private static LazyInitializedSingleton instance;

    private LazyInitializedSingleton() {
    }

    public static LazyInitializedSingleton getInstance() {
        if (instance == null) {
            instance = new LazyInitializedSingleton();
        }
        return instance;
    }
}

4. Thread safe Singleton

Only with ‘synchronized’ key word, only one thread can execute this method at a time.


class ThreadSageSingleton {
    private static ThreadSageSingleton instance;

    private ThreadSageSingleton() {
    }

    public static synchronized ThreadSageSingleton getInstance() {
        if (instance == null) {
            instance = new ThreadSageSingleton();
        }
        return instance;
    }

    // above make sure the thread-safe but doesn't have good performance
    // because of the cost associated with the synchronized method
    // we need it only for the first few threads who might create the separate instances

    // so the double check could solve this ↓
    public static synchronized ThreadSageSingleton getInstanceWithCondition() {
        if (instance == null) {
            synchronized (ThreadSageSingleton.class) {
                if (instance == null) {
                    instance = new ThreadSageSingleton();
                }
            }
        }
        return instance;

5. Reflection will destroy singleton

since it create a new instance by the reflect constructor


class ReflectionSingletonTest {

    public static void main(String[] args) {
        EagerInit instanceOne = EagerInit.getInstance();

        try {
            Constructor[] constructors = EagerInit.class.getDeclaredConstructors();
            for (Constructor constructor : constructors) {
                //Below code will destroy the singleton pattern
                constructor.setAccessible(true);
                break;
            }
        } catch (Exception e) {
            e.printStackTrace();
        }

        EagerInit instanceTwo = null;

        try {
            Constructor[] constructors = EagerInit.class.getDeclaredConstructors();
            for (Constructor constructor : constructors) {
                //Below code will destroy the singleton pattern
                constructor.setAccessible(true);
                instanceTwo = (EagerInit) constructor.newInstance();
                break;
            }
        } catch (Exception e) {
            e.printStackTrace();
        }


        System.out.println(instanceOne.hashCode());
        System.out.println(instanceTwo.hashCode());
    }

}

6. The best practice of singleton in Java is enum

which has all the features we need,
like instantiated only once, globally accessible
but drawback is not support lazy load


enum EnumSingleton {
    INSTANCE;
    public static void dosth() {}

}

7. Serialized Singleton

sometime, in distributed systems, we need implement serializable interface in singleton
so we could store its state in file system and future retrieve


class SerializedSingleton implements Serializable {

    private static final String serialVersionUID = UUID.randomUUID().toString();

    private static SerializedSingleton instance;

    private SerializedSingleton() {}

    public static SerializedSingleton getInstance() {
        if (instance == null) {
            synchronized (SerializedSingleton.class) {
                if (instance == null) {
                    instance = new SerializedSingleton();
                }
            }
        }
        return instance;
    }

    protected Object readResolve() {
        return getInstance();
    }

}

Test Class


class SingletonSerializedTest {
    public static void main(String[] args) throws IOException, ClassNotFoundException {
        SerializedSingleton instanceOne = SerializedSingleton.getInstance();
        ObjectOutput out = new ObjectOutputStream(new FileOutputStream("recordFile.ser"));
        out.writeObject(instanceOne);
        out.close();

        // deserialize
        ObjectInput in = new ObjectInputStream(new FileInputStream("recordFile.ser"));
        SerializedSingleton instanceTwo = (SerializedSingleton)in.readObject();
        in.close();

        System.out.println(instanceOne.hashCode());
        System.out.println(instanceTwo.hashCode());
    }
}
