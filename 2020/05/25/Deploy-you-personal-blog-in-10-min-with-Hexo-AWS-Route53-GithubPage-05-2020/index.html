十分钟搭建个人博客(Hexo + GithubPage + AWS Route53) [05.2020]


前言

疫情期间保持学习节奏，决定利用博客来记录和总结学习的点滴进度。
在网上看到了Hexo是个很不错的静态页面生成框架，之前浏览其他大佬的页面时候也看到过这个名字，想拿来试试
但是鉴于能Google到的教程是在太古老，导致写Blog5分钟，部署搭环境两小时
决定用个人博客的第一篇来share，在时至今日(2020年 5月)用Hexo搭建博客中踩过的坑与注意事项
希望能够帮助到看到这篇文章各位


大致流程：

(1) Hexo环境配置 -> Hexo本地创建静态页面 -> 本地起服务测试
(2) 链接Github仓库，利用Github Page测试线上页面
(3) 个性化域名连接(以AWS Route 53为例)



1. Hexo前提环境配置

有npm基础的同学应该已经安装好了 可以略过到2
(1) 安装Git


Mac: 使用 Homebrew 或 MacPorts 比较方便
Windows: 下载并安装 git
Linux (Ubuntu, Debian)：sudo apt-get install git-core
Linux (Fedora, Red Hat, CentOS)：sudo yum install git-core

(2) 安装Node.js


Mac：依然是 Homebrew 或 MacPorts 方便些。
Windows：通过 nvs（推荐）或者nvm 安装。
Linux（DEB/RPM-based）：从 NodeSource 安装。

2. Hexo安装

还不够熟悉npm操作的同学可以全局安装
npm install -g hexo-cli
有可能会permission deny, 请在命令前加sudo 然后输入账户密码即可


足够熟悉的同学可以局部安装hexo
npm install hexo

3. Github创建仓库与Github Page

如果还没有Github账户的同学请先 注册,用户名尽量全部小写

(1) 新建项目
！注意:项目名必须以你的用户名.github.io命名
如:username.github.io

格式按图片上的yourusername改成你的username即可

原因是因为Github page生成网页会根据你的repo项目命名来生成，如果不是用户名.github.io的格式url格式会变 从而导致链接你的自定义域名失败

(2) Github Page
通过创建初始readme.md或者后面push上去初始的hexo文件后
点击仓库的Setting页面

向下拉会有GitHub Pages一栏。
下方会有刚才的用户名.github.io显示，
点击选择master分支，主题可选可不选

稍等一会儿等到链接处告诉你ready变绿

即可点击页面看到GithubPage初始页面

！注意:这里如果在新建repo时没点新建README文件，选择主题后commit一个changes会显示对应主题的index页面，如果点了readme则可能是一个白色页面只有一个title
这两种情况都可以
初学同学推荐不选择主题 不然后面还需要处理conflict

4. Hexo创建本地项目

推荐在一个新的干净文件夹下存放Hexo生成的本地项目文件和博客目录比如’/Users/xxx/Projects/HexoBlog’
在之前步骤安装成功的标志是hexo -v或者npx hexo -v能看到Hexo所需相关依赖的版本号


确认安装成功后用hexo init在一个空文件夹下创建初始文件
然后使用npm i安装npm的依赖包 (依然会有权限问题 老样子sudo解决)

然后使用hexo g/npx hexo g生成静态页面
注：这个命令每次要生成新post后都会需要
再用hexo s/npx hexo s启动本地服务
打开 http://localhost:4000/ 就可以看到初始博客效果了


5.链接Github仓库与本地Hexo repo

有git基础的同学可略过本部分
(1) 在本地的刚建好的hexo文件目录下初始git
git init git add . git commit -m "init"

(2) 然后链接远程仓库
git remote add orign + 你的仓库url 比如我用来测试的
git remote add origin https://github.com/Martin7374/martin7374.github.io.git

(3)之后拉取并处理冲突
如果选择主题则需要在pull加上–allow-unrelated-histories
git pull origin master --allow-unrelated-histories
如果之前没选择应该直接
git pull origin master
就可以

然后再民工三连提交本地hexo代码到github
git init git add . git commit -m "init"
此时稍等一下再打开你的username.github.io应该能看到本地hexo的页面部署到了云端！


6.自定义域名链接(以AWS Route 53为例)

此时如果觉得github.io结尾域名已经满足需求可以到此为止装逼去了
如果觉得还不够可以看完这最后一步

进入你用的云平台域名管理页面 链接你的github page域名 国内同学阿里云，xx云原理类似 可以寻找对应教程 不必完全类比

这里以AWS Route 53为例
购买好你想要的域名后打开Route53
点击Hosted zones
点击你的域名
